# Beget и API: что реально можно сделать

## Важно про выкладку сайта

**Официальный Beget.API хостинга не умеет заливать содержимое файлов сайта** (нет метода «положить index.html в public_html»). Через API доступны, например: сайты/домены, DNS, БД, почта, бэкапы, крон и т.д. См. обзор: [Beget.API хостинга](https://beget.com/ru/kb/api/beget-api).

Загрузка статики на сервер по-прежнему делается так:

- файловый менеджер в панели;
- **FTP / SFTP / SSH** (после настройки доступа).

Отдельно есть API блока **управления FTP-аккаунтами** (список / добавить / смена пароля / удалить), но **не замена FTP**: после создания аккаунта файлы всё равно передаются по FTP/SFTP. См.: [Управление FTP по API](https://beget.com/ru/kb/api/funkczii-upravleniya-ftp).

Облачные сервисы Beget — другой API ([developer.beget.com](https://developer.beget.com/)), к обычному виртуальному хостингу «просто залить сайт по HTTP» он не подставляется.

## Как включить доступ по API к хостингу

1. Панель [cp.beget.com](https://cp.beget.com).
2. Задай **отдельный пароль для API** (в базе знаний это описано в разделе про API: [api.beget.com](https://api.beget.com/) — принципы работы, параметры `login` и `passwd`).

Дальше можно вызывать методы по HTTPS из своих скриптов.

## Итог для этого проекта

**Автовыкладка «только Beget API без FTP/SSH» для статического сайта не предусмотрена.**  
Реалистичные варианты: вручную через файловый менеджер, по FTP после создания пользователя (при желании — через API `ftp/add`), или по SSH/rsync после настройки ключа.

Ранее использовавшийся GitHub Actions + SSH мы убрали по твоей просьбе; при необходимости его можно вернуть отдельно.
